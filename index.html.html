Publicar en GitHub Pages
GitHub Pages convierte este archivo HTML en una URL pública que funciona desde cualquier dispositivo. Es gratis y el deploy es automático cada vez que subes el archivo.
Paso a paso
1
Ve a github.com y crea una cuenta gratuita si no tienes una. Luego haz clic en New repository.
2
Ponle nombre al repo, por ejemplo originations-dashboard. Asegúrate de que sea Public. Marca la opción Add a README file y haz clic en Create repository.
3
Dentro del repo haz clic en Add file → Upload files. Arrastra el archivo originations_dashboard_v3.html y haz clic en Commit changes.
4
Ve a Settings → Pages (en el menú lateral izquierdo). En Source selecciona Deploy from a branch, elige main y la carpeta / (root). Haz clic en Save.
5
Espera ~1 minuto y actualiza la página. GitHub mostrará tu URL, algo como: https://tu-usuario.github.io/originations-dashboard/originations_dashboard_v3.html. ¡Listo!
Cada semana — actualizar el dashboard
A
Agrega tus nuevos reviews con el botón ⊕ Pegar desde Sheets o manualmente en Base de Datos.
B
Haz clic en Exportar CSV para tener un backup, y descarga el archivo HTML actualizado con Guardar página como en tu navegador (Ctrl+S / Cmd+S).
C
En GitHub ve a tu repo, haz clic en el archivo, luego en el ícono de lápiz ✏️ o usa Add file → Upload files para reemplazarlo. El deploy es automático en ~1 minuto.
⚠️ Los datos se guardan en localStorage del navegador. Al subir el archivo a GitHub los datos no viajan con él — solo el código. Siempre agrega los reviews desde el navegador donde tienes el dashboard abierto.
Pegar desde Google Sheets
Selecciona las filas nuevas en tu Google Sheet (incluyendo el encabezado si quieres), cópialas con Ctrl+C y pégalas abajo con Ctrl+V. El dashboard detecta las columnas automáticamente.
